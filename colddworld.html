ColddWorld: Immersion Vulnhub Walkthrough

Level : Beginner / Intermediate

Goal: Find user flag and root flag

Summary

This is a writeup for the Vulnhub Machine - ColddWorld: Immersion which was released by C0ldd

Machine Discovery

First we will use nmap -sn 192.168.0.0/24 to discover the machine in our network. (I have assigned an IP of 192.168.0.0/24 range to my machine)

Port Scanning

Once the machine is discovered, we will do a port scan using nmap

nmap -A -T4 -p- 192.168.0.113

Exploiting User Flag

We can see that Port 80(HTTP) and Port 3042(SSH) are open
Lets start with Port 80
Open the website and you will see nothing but a single page
We need to search for hidden directories. I have used Gobuster for directory bruteforcing

gobuster dir -u http://192.168.0.113/ -w /usr/share/dirbuster/wordlists/directory-list-2.3-medium.txt -x php,php3,html,txt

We can see some directories like /login, /wp, /css, /secure, /js
We need to move forward with /login, so enter the URL in the browser and you will see a login page

192.168.0.113/login

If we enter anything, it redirects us to a blank page

Now go back and check the source code of /login page

We found two users c0ldd and carls. C0ldd has a left a message that tells us about login credentials of carls stored in carls.txt in /var directory
Now login with any credentials and notice the URL
This URL is vulnerable to Local File Inclusion(LFI)
So lets navigate two directories back to /var directory and find the carls.txt

192.168.0.113/login/account.php?page=../../../carls.txt

We got the password of carls, but this password is base64 encoded so lets decode it

echo “Y2FybG9z” | base64 -d

We got the password as carlos
Now lets login to SSH with username carl and password carlos on port 3042

ssh 192.168.0.113 -l carls -p 3042

If you cd in carls you won’t find anything but we can find another user c0ldd in the /home directory
We need to login as c0ldd to process further
Lets use sudo -l to search for programs which we can run as c0ldd

sudo -l

We can see /bin/bash command for user c0ldd which we can execute as carls
Lets try it

sudo -u c0ldd bash

Exploiting Root Flag

We logged in as c0ldd, now lets navigate into his /home/c0ldd directory
We can see two files, lets first open user.txt
This is base64 encoded so lets decode it and we got our User Flag!
Now we need to login as root so lets use sudo -l

sudo -l

We can run a python script DoNotRun.py which was included in the home directory of c0ldd
If we open it you will see a sample python code of hello world
We need to change this script and write a script to execute a bash
If you try to modify it, you won’t have to permission. So lets delete it and create a new DoNotRun.py
I used the below python script to open bash

nano DoNotRun.py

Now lets execute this script as root

sudo -u root /usr/bin/python3 /home/c0ldd/DoNotRun.py

And we got access to root!
Now navigate to the /root directory and open the root.txt
It is base64 encoded so decode it and we got our root flag!
